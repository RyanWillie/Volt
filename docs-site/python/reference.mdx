## Design root

| Export | Purpose |
| --- | --- |
| `volt.Design` | Root handle for one kernel-owned logical circuit plus schematic and PCB projections. |

Common methods: `net`, `nets`, `R`, `C`, `CP`, `L`, `LED`, `diode`, `switch`, `crystal`, `test_point`, `connector_1x01`, `connector_1x02`, `connector_1x03`, `regulator`, `op_amp`, `define_component`, `define_module`, `instantiate`, `schematic`, `board`, `validate`, `validate_for_pcb`, `validate_bom_readiness`, `set_sourcing_snapshot`, `bom`, `bom_csv`, `to_json`, and `write`.

## Project framework

| Export | Purpose |
| --- | --- |
| `volt.BuildContext` | Read-only access to earlier stage models when a project stage needs to address multiple designs. |
| `volt.ProjectResource` | Explicit authoring resource produced by one stage for later build stages. |
| `volt.Project` | Staged workflow entry point for design, schematic, PCB, diagnostics, tests, and bundle writing. |
| `volt.ProjectStage` | Stage handle used as a build decorator and result lookup key. |
| `volt.ProjectResult` | Output of one project run with collected models, diagnostics, tests, and artifact writing. |
| `volt.ProjectArtifactPaths` | Paths written by project-owned example and viewer artifact export. |
| `volt.ManufacturingPackageResult` | Summary returned by `ProjectResult.write_manufacturing_package` after writing a deterministic manufacturing handoff package. |
| `volt.ManufacturingPackageError` | Refusal raised when a manufacturing package would be unsafe or ambiguous to write. |
| `volt.ExpectedDiagnostic` | Project-local declaration for a known diagnostic expected in examples or benchmarks. |
| `volt.ExpectedDiagnosticResult` | Match status for one expected diagnostic declaration after a project run. |
| `volt.ProjectDiagnostics` | Ordered project-level diagnostics across default model validators. |
| `volt.ProjectDiagnostic` | One diagnostic annotated with project stage, source, and report metadata. |
| `volt.ProjectTestDefinition` | Registered product-intent test attached to one stage. |
| `volt.ProjectTestResult` | Pass/fail result for one stage-attached project test. |
| `volt.StageResult` | Summary of one executed stage and its project-test results. |

## Diagnostics

| Export | Purpose |
| --- | --- |
| `volt.DiagnosticReport` | Iterable validation report with `has_errors`. |
| `volt.Diagnostic` | One diagnostic severity, category, code, message, entity list, optional overlays, and optional measurement. |

| `volt.PART_LINEUP_DIAGNOSTIC_CODES` | Stable part lineup diagnostic code strings exported for library and project checks. |
