555 timer PCB benchmark

This example turns the existing Volt-native 555 LED blinker schematic into a routed real-board PCB benchmark using public Python APIs over the kernel-owned PCB model.

Circuit NE555 astable LED blinker with timing RC network, control bypass, LED load, and power header.
Board Two copper layers, silkscreen, rectangular outline, mounting holes, placements, tracks, and vias.
Validation Logical, schematic, PCB-readiness, and board DRC reports must all serialize cleanly.

Why this board exists

VOL-161 needs one deterministic, routed, real-board PCB example that exercises the public Python authoring facade without making Python the owner of EDA semantics. The 555 timer is the better benchmark because the logical and schematic example already exists, so the PCB layer proves that the same circuit can be carried through physical part selection, footprint resolution, placement, routing, DRC, JSON, and SVG.

Routing authoring

Hand-placed copper is expressed from board anchors using layout.route(), .tox(), .toy(), directional jogs, and layout.via(). The route builder only describes physical implementation intent; it still lowers through the kernel-owned board track and via APIs.

Boundary

The Python example describes author intent and calls kernel-backed mutation APIs. Logical connectivity remains in the circuit model. The schematic only presents that connectivity, and the PCB only implements it physically through selected parts, footprint mappings, board geometry, copper, and validation artifacts.
