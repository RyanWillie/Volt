The STM32 USB buck example shows how Volt scales beyond a single-file toy design while keeping kernel ownership clear.

Source: `examples/stm32_usb_buck/`.

## What it demonstrates

- Reusable logical modules for power, USB, support circuitry, and LEDs.
- A small library in `python/volt/libraries/stm32_usb_buck.py`.
- Module instance port binding into parent nets.
- Custom schematic symbols for MCU, USB, passives, connectors, and regulators.
- Regional schematic authoring for power, MCU, and connector areas.
- Generated logical JSON, schematic JSON, SVG, per-page SVG, and validation report JSON.

## Project shape

| File | Responsibility |
| --- | --- |
| `stm32_board.py` | Top-level logical composition. |
| `power_blocks.py` | Power input and regulator module. |
| `connection_blocks.py` | USB and connector modules. |
| `utility_blocks.py` | External supply and LED helper modules. |
| `schematic_output.py` | Shared sheet metadata. |
| `schematic_power.py` | Power-region drawing. |
| `schematic_mcu.py` | MCU-region drawing. |
| `schematic_connectors.py` | USB and connector-region drawing. |

## Pattern to copy

Keep logical composition separate from schematic projection:

```python
@project.design
def design():
    stm32 = build_board()
    return stm32.design, volt.ProjectResource("stm32_board", stm32)

@project.schematic
def schematic(context: volt.BuildContext) -> volt.Schematic:
    board = context.resource("stm32_board", Stm32UsbBuckBoard)
    nets = {net.name: net for net in board.design.nets()}
    sheet = board.design.schematic("STM32 USB Buck", **SHEET_OPTIONS)

    power_region = sheet.region("Power Circuitry", x=18, y=18, w=558, h=116)
    mcu_region = sheet.region("STM32 Microcontroller", x=18, y=140, w=346, h=266)
    connectors_region = sheet.region("Connectors and USB", x=370, y=140, w=208, h=216)

    _author_power_region(power_region, board, nets)
    _author_mcu_region(mcu_region, board, nets)
    _author_connectors_region(connectors_region, board, nets)
    return sheet
```

That separation makes it easier to test logical correctness independently from schematic readability while still passing stable stage inputs through the Project Framework.
