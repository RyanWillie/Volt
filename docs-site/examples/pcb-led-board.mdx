The PCB LED board is the first full path from logical authoring to a board projection.

Source: `examples/pcb_led_board/main.py`.

## Flow

<Steps>
  <Step title="Create the logical circuit">
    Define `+3V3`, `LED_A`, and `GND`, then connect a header, resistor, and LED.
  </Step>
  <Step title="Select physical parts">
    Attach manufacturer, part number, package label, object-owned footprint geometry, pin-pad mappings, and a small `model_3d` asset reference.
  </Step>
  <Step title="Carry footprint geometry">
    Keep the through-hole and 0603 pad geometry on each selected physical part.
  </Step>
  <Step title="Place and route">
    Add layers, outline, mounting holes, placements, and tracks.
  </Step>
  <Step title="Validate and write artifacts">
    Run logical, PCB-readiness, schematic, readability, and board validation before writing JSON/SVG/KiCad artifacts.
  </Step>
</Steps>

## Selected part example

```python
from pathlib import Path

MODEL_DIR = Path(__file__).resolve().parent / "models"

parts["R1"].select_part(
    manufacturer="Yageo",
    part_number="RC0603FR-07330RL",
    package="0603",
    footprint=volt.FootprintDefinition(
        ("passives", "R_0603_1608Metric"),
        pads=(
            volt.FootprintPad.surface_mount("1", at=(-0.75, 0.0), size=(0.80, 0.95)),
            volt.FootprintPad.surface_mount("2", at=(0.75, 0.0), size=(0.80, 0.95)),
        ),
    ),
    pin_pads={1: "1", 2: "2"},
    model_3d=volt.PartModel3D(
        MODEL_DIR / "r_0603_body.step",
        offset=(0.0, 0.0, 0.35),
    ),
    power_rating=0.1,
)
```

The full project bundle copies the STEP asset into `assets/models/` and writes
`assets/part_models_3d.json` plus a board placement map, while the logical JSON stores
only the selected-part model metadata.

## Board example

```python
board = design.board("First Board LED")
front = board.add_layer("F.Cu", role="copper", side="top")
back = board.add_layer("B.Cu", role="copper", side="bottom")
board.set_layer_stack((front, back), thickness=1.6)
board.set_rectangular_outline(origin=(0.0, 0.0), size=(32.0, 18.0))

with board.layout(unit=1.0) as layout:
    header = layout.place(parts["J1"], at=board.edge("left").center().right(5.0))
    resistor = layout.two_pad(parts["R1"]).at((15.0, 7.0)).anchor("center").right()
    layout.connect(header[1], resistor[1], layer=front, width=0.25, mode="direct")
```

## KiCad handoff

```python
result = run_project()
export = result.board().to_kicad_pcb()
assert export.warnings == ()
```

The KiCad file is an adapter output for inspection and downstream Gerber/Excellon generation. Volt board JSON remains the source of truth. The checked-in artifact is expected to parse in KiCad, pass KiCad PCB DRC, and export Gerber plus Excellon drill files.

This is the best starting point for learning how Volt connects logical selected parts to physical footprint and pad data without using the board API to choose library part geometry.
