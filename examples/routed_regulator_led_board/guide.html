Routed regulator LED board

This routed real-board PCB benchmark proves a compact Volt design can move from logical circuit authoring to schematic artifacts, routed PCB state, DRC, and previews using public Python APIs over the kernel-owned PCB model.

What It Exercises

  • Power header, regulator IC, capacitors, LED resistor, and indicator LED.
  • Selected physical parts with resolved footprints for every placed component.
  • Board outline, two-layer copper stack, mounting holes, silkscreen, tracks, and a via.
  • Deterministic logical, schematic, PCB JSON, SVG previews, and validation output.

Architecture Boundary

Logical connectivity stays canonical in the circuit. PCB routing implements existing nets with generic board primitives; the benchmark does not hand-edit generated JSON or SVG.

Generated Artifacts

  • routed_regulator_led_board.volt.json
  • routed_regulator_led_board.volt.schematic.json
  • routed_regulator_led_board.svg and page/body SVGs
  • routed_regulator_led_board.volt.pcb.json
  • routed_regulator_led_board.pcb.svg
  • routed_regulator_led_board.validation.json

Out of scope

Autorouting, fabrication export, manufacturing sign-off, product UI, 3D editing, and the larger STM32 USB buck board remain outside this first routed board benchmark.
