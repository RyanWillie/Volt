VOL-215 Part Lineup Contract Validation

Goal: make pin-map, schematic symbol, and footprint projection drift mechanically impossible at artifact assembly/load boundaries, while reporting design-quality lineup concerns through shared diagnostics.

Architecture

The C++ part artifact remains the owner of EDA meaning. VOL-214 stored hash references for symbols and footprints; this slice adds the minimum projection summaries needed for validation: symbol pin occurrences and footprint pad summaries. Python continues to author ergonomic volt.Part objects and lowers them into the kernel artifact/diagnostic API.

Tasks

  1. Red tests for structural rejection. Add C++ tests for part-constructor assembly and artifact-reader load rejection: symbol pin outside pin map, missing/duplicated symbol coverage, pin-pad mappings to unknown pins or pads, collapsed multi-pad mappings, and repeated pin names with non-distinct symbol resolution.
  2. Red tests for diagnostics. Add C++ tests for PART_PIN_WITHOUT_PAD, PART_PAD_WITHOUT_PIN, PART_PAD_OVERLAP, PART_PAD_ROW_PITCH_INCONSISTENT, plus a correct part with zero lineup diagnostics.
  3. Kernel implementation. Extend PartDefinition, symbol references, and orderable footprint references with projection summaries; enforce structural rules in constructors and reader; add shared part diagnostic codes/category and validate_part_lineup.
  4. Python bridge. Lower Python symbol pins and footprint pads into the kernel payload; expose part diagnostic codes and lineup diagnostics from the binding; map kernel diagnostics into LibraryResult without turning design-quality warnings into artifact rejection.
  5. Determinism and verification. Update the golden volt.part fixture and content hash, run targeted C++ and Python tests, then run the broader available checks before commit.
  6. Publish and review. Commit, push feat/vol-215-part-lineup-contract-validation, open a ready PR, create the requested thermonuclear review thread, post review findings on the PR, and address actionable feedback.
