Volt authoring guide

SchemDraw-style grammar, Volt-owned semantics.

Use the drawing sugar to place a readable schematic quickly, while logical nets, pins, components, validation, JSON, and SVG remain owned by the Volt kernel.

Read this once before authoring examples or documentation. The boundary is the important part.
SchemDraw is an inspiration, not a dependency. Volt borrows the pleasant cursor grammar and visual quality target. It does not delegate EDA meaning to SchemDraw or to Python-only state.
First

Author the logical circuit

Create components, nets, pin membership, intentional stubs, and logical no-connect intent before drawing.

Then

Project the schematic

Place symbols, route wires, add labels, ports, and markers. These objects visualize existing kernel state.

Verify

Validate readiness

Run schematic validation to catch missing pin coverage, weak no-connect presentation, and fragmented visual nets.

Minimal resistor and LED flow. Full source: examples/schematic_sugar/compact_led.py.
design = volt.Design("sugar-led")
vcc = design.net("+3V3", kind="power", voltage=3.3)
sig = design.net("LED_A")
gnd = design.net("GND", kind="ground")
r1 = design.R("330 ohm", ref="R1")
d1 = design.LED(ref="D1")

vcc += r1[1]
sig += r1[2], d1["A"]
gnd += d1["K"]

sheet = design.schematic("Main")
with sheet.drawing(unit=20) as d:
    r = d.two_terminal(r1).right().label_ref().label_value()
    led = d.two_terminal(d1).right().reverse().label_ref()

    d.connect(r.end, led.start)
    d.power("+3V3", at=r.start)
    d.ground(at=led.end)

The LED is reversed for presentation so the anode anchor is on the left and the cathode anchor is on the right. That changes the symbol orientation only; the logical netlist was already authored above.

The cursor is local to one drawing session. It stores only presentation position and direction.

here

The current sheet point. Two-terminal placement starts here and advances the cursor to the dropped endpoint.

direction

One of Right, Down, Left, or Up. Placement helpers use it unless overridden.

unit

The drawing distance used by .right(), .down(), and other unit-based element placement calls.

Two-terminal elements are sugar over normal symbol placement.

Common passives

Use d.two_terminal(component) for exactly two-pin components with default symbols, regardless of whether the component is a resistor, capacitor, inductor, diode, LED, or another two-terminal element.

Anchors

The returned handle exposes start, end, center, pin numbers, and unique pin names. These are schematic anchors tied back to kernel pins.

with sheet.drawing(at=(20, 20), unit=20) as d:
    c_in = d.two_terminal(cin).down().label_value()
    d.move_from(c_in.start.right(60))
    c_out = d.two_terminal(cout).down().label_value()
Net inference is intentionally conservative.
Inference never creates connectivity. d.connect(a, b) can infer a net only when the endpoints already prove the same logical net in the kernel.
Call When it works When to pass net=
d.connect(pin_a, pin_b) Both pin anchors belong to the same logical net. If either endpoint is a plain coordinate or a non-pin port.
d.power("+3V3", at=pin) The pin anchor is already connected to +3V3. If the port is placed away from the pin, such as at=pin.up(20).
d.off_page("SWDIO", at=point) The name matches an existing logical net and the point is inferable. When routing to a label or coordinate: d.off_page("SWDIO", net=swdio, at=...). 
d.connect(regulator.OUT, output_cap.start, shape="-")
d.connect(regulator.GND, ground_port, net=gnd, shape="|-", k=10)
Labels and ports are schematic projection data, not alternate netlist data.

Reference and value

.label_ref() and .label_value() emit symbol fields. Generic .label(...) is for local presentation text.

Net labels and ports

d.net_label(...), d.power(...), d.ground(...), and d.off_page(...) must resolve to existing logical nets.

No-connect markers

d.no_connect(pin) draws a marker. Use pin.mark_no_connect() separately when the logical design intent is that the pin is unused.

Use generic IC/block symbols when the readable shape matters more than a prebuilt library symbol.
The builder emits ordinary Volt symbol data. Side, slot, lead, and label choices are Python authoring vocabulary that lower to durable symbol pins and primitives. 
timer_symbol = volt.SchematicSymbolSpec.ic(
    "examples:NE555",
    pins=(
        volt.SchematicSymbolSpec.ic_pin("DISCH", 7, side="left", slot=1),
        volt.SchematicSymbolSpec.ic_pin("TRIG", 2, side="left", slot=2),
        volt.SchematicSymbolSpec.ic_pin("OUT", 3, side="right", slot=2),
        volt.SchematicSymbolSpec.ic_pin("VCC", 8, side="top", slot=4),
    ),
    center_label="555",
    bottom_label="timer",
)

Repeated connector and IC fanouts should still be explicit routes over existing nets.

ortho_lines() is a loop-saving helper for local fanouts. Each entry is either (start, end), where Volt infers the already-shared net, or (net, start, end) for coordinate and port endpoints.

d.ortho_lines(
    (
        (timer.OUT, led_resistor.start),
        (timing, timer.THRESH, timing_cap.start),
        (timing, timer.TRIG, timing_cap.start),
    ),
    shape="-|",
    k=-18,
)
Common authoring failures and the fastest fixes.
Message pattern Meaning Fix
different logical nets You tried to draw one visual wire between pins that the logical circuit says are separate. Fix the logical net membership, or route each net separately.
explicit net or non-pin anchor A coordinate, offset anchor, or port does not prove a kernel net by itself. Pass net=existing_net and keep the logical circuit unchanged.
No default schematic symbol The component definition has no durable default symbol for placement. Use a component with a default symbol, pass symbol=, or define kernel-owned symbol metadata on the component definition.
SCHEMATIC_PIN_NET_NOT_VISUALLY_COVERED A connected logical pin is not visibly covered by a wire, label, port, or compatible schematic object. Add a route or port that touches the placed pin anchor.
SCHEMATIC_NO_CONNECT_INTENT_NOT_MARKED The logical circuit marks a pin intentionally unused, but the schematic does not show it. Add d.no_connect(pin, reason=...) at the placed pin anchor.
Do not patch meaning into the schematic. If a wire fails because the logical nets differ, change the logical circuit first. The schematic layer is a projection surface.
