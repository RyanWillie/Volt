VOL-162 handoff contract

KiCad PCB is a manufacturing bridge, not Volt's board model.

Volt board JSON and kernel board data stay canonical. The KiCad writer consumes that data and emits a deterministic .kicad_pcb file for inspection and downstream fabrication tooling.

Boundary

Export is one-way and adapter-shaped.

The writer lives under volt/adapters/kicad, depends on Volt circuit and PCB model targets, and does not make KiCad file structures part of core, IO, or board mutation APIs.

Logical nets, selected parts, footprint geometry, placements, copper routes, board text, and board outline come from Volt-owned data. The exporter never creates, merges, splits, infers, or repairs logical connectivity.
Supported subset

The first subset is intentionally narrow.

Volt construct KiCad output Current support
Board outline gr_line edges on Edge.Cuts Polygon outlines export as deterministic closed edge segments.
Layers KiCad layer table Top and bottom copper layers are mapped to F.Cu and B.Cu. Standard paste, mask, silkscreen, fabrication, and edge-cut layers are emitted as adapter output layers when needed by KiCad.
Footprints, pads, placements Inline KiCad footprint and pad blocks Top-side placements export with surface-mount, through-hole, and non-plated mechanical pads. Generated reference/value metadata is placed on fabrication layers so it does not become copper. Pad nets are taken only from Volt pad resolution.
Mounting holes Non-plated through-hole footprint Board hole features export as drill-producing KiCad footprints.
Tracks segment Routed point chains export as one KiCad segment per adjacent point pair on mapped copper layers.
Vias via Vias export between mapped copper layers using Volt drill and annular diameters.
Text and annotations gr_text Board text exports on mapped KiCad layers with deterministic size, position, and rotation.
Zones and keepouts Loss report only Not exported in the first subset. They are fab-critical losses because copper pours and constraints can change the manufacturable board.
Nets KiCad net table plus pad, route, and via references Net names and IDs come from the canonical Volt circuit. The exporter does not synthesize nets.
Python handoff

Call the adapter from a board projection.

export = board.write_kicad_pcb("artifacts/pcb_led_board.kicad_pcb")
if export.warnings:
    for warning in export.warnings:
        print(warning.fabrication_impact, warning.construct, warning.message)

The example board at examples/pcb_led_board commits a generated pcb_led_board.kicad_pcb artifact alongside Volt JSON and SVG output. Project runs collect fab-critical KiCad export losses as PCB_KICAD_FAB_EXPORT_LOSS diagnostics under pcb.kicad_export.

Loss gating

Losses are classified before Python sees them.

Impact Examples Project behavior
fab-critical Unsupported copper zones, keepouts, slots, cutouts, plated board holes, omitted component placements, missing footprints, invalid pin-pad mappings, tracks or vias on unmapped copper, and text on fabrication-facing unmapped layers. Converted to an error diagnostic with code PCB_KICAD_FAB_EXPORT_LOSS. The diagnostic rule identity is the KiCad loss construct, such as board.zone or board.feature.slot, so expected-diagnostics policy can explicitly allow only the known loss.
informational Annotation text on a mechanical/documentation Volt layer that has no KiCad PCB layer mapping. Remains in KiCadPcbExport.warnings for visibility, but does not enter the standard project diagnostics gate.
A clean badge-class routed board should export with no fab-critical losses. The kicad_routed_badge.kicad_pcb golden fixture pins byte-stable ordering, units, coordinates, pad counts, track segments, vias, and net-table output.
Inspection

Use KiCad as the inspection and fabrication tool.

  1. Open the generated .kicad_pcb in KiCad PCB Editor.
  2. Inspect board outline, drill holes, footprints, pad nets, routed tracks, and vias.
  3. Run KiCad DRC for fabrication-facing checks that are outside Volt's current subset.
  4. Use KiCad's plotting tools to generate Gerber and Excellon files for a fabricator.
python3 scripts/verify-kicad-fab-export.py artifacts/pcb_led_board.kicad_pcb --output-dir artifacts/kicad-fab --drc

The script runs kicad-cli pcb export gerbers, drill export, and optional kicad-cli pcb drc when KiCad is installed. It exits successfully with a clear skip message when kicad-cli is absent, so CI does not depend on KiCad.

Treat adapter warnings as part of the artifact. Fab-critical warnings mean the KiCad handoff omitted board meaning that can affect fabrication; resolve the construct or explicitly accept the diagnostic before manufacturing from the KiCad file.
