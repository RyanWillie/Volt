Architecture decision · Issue #261

A small typed Circuit aggregate

Volt exposes one authoritative logical aggregate whose public surface is complete typed specifications, irreducible graph operations, typed updates, generic reads, and free derived queries.

Accepted Pre-1.0 breaking change Python surface preserved No semantic loss

Core decision

Circuit remains the sole canonical logical model, but its public API will no longer mirror its normalized storage tables.

Internal stores may remain decomposed. Public callers see complete domain operations and typed inspection—not connectivity, hierarchy, electrical, intent, or net-class mutation facades.

≈15intended root declarations, defended as a budget
1canonical logical authority and invariant boundary
0public storage facades, raw restore hooks, or generic entity mutation handles

Why the current split is being replaced

The former facades grouped names, not ownership

Each former mutator stored a borrowed Circuit& and forwarded into the same aggregate. It owned no independent state or invariant boundary, so the conceptual model remained one large storage-shaped API.

The storage model leaked into authoring

Incremental pin/module table insertion, public loader restoration, count-plus-ID traversal, and public query models expose implementation mechanics rather than stable electronics operations.

The lifetime model became less safe

A facade obtained from a temporary root could dangle. #234 closed that live hazard before the facades were removed.

The surface grows with every internal store

The accepted evolution rule reverses that pressure: new storage stays private; public growth requires a new irreducible domain operation.

Ownership model

AuthoringPython · C++ · importers
→
Circuitcanonical graph · typed meaning · invariants
→
Consumersvalidation · schematic · PCB · BOM · export
  • Circuit owns definitions, instances, pins, nets, connectivity, hierarchy, typed attributes, selected parts, assembly intent, net classes, and explicit logical intent.
  • Schematic and PCB own presentation and physical implementation data, but never logical connectivity.
  • Python owns ergonomic syntax and handles, but lowers every meaningful operation into the kernel.
  • Validation and analysis own read-only computation and diagnostics, not model state.

Target public API

This is the architectural shape. Strong type names may be refined during implementation, but categories may not expand casually.

class Circuit final {
  public:
    // Complete reusable definitions.
    ComponentDefId define_component(ComponentSpec);
    ModuleDefId    define_module(ModuleSpec);
    NetClassId     define_net_class(NetClassSpec);

    // Canonical graph construction.
    ComponentId      instantiate_component(ComponentDefId, ComponentInstanceSpec);
    ModuleInstanceId instantiate_module(ModuleDefId, ModuleInstanceSpec);
    NetId             add_net(NetSpec);

    // Irreducible structural operations.
    bool          connect(NetId, PinId);
    bool          disconnect(PinId);
    PortBindingId bind_port(ModuleInstanceId, PortDefId, NetId);

    // Progressive, strongly typed meaning.
    void update(ComponentId, ComponentUpdate);
    void update(NetId, NetUpdate);
    void mark_no_connect(PinId);

    // Generic fundamental reads.
    template <typename Id> requires CircuitEntityId<Id>
    const entity_type_t<Id>& get(Id) const;

    template <typename Id> requires CircuitEntityId<Id>
    entity_range_t<Id> all() const;

    std::optional<NetId> net_of(PinId) const;
};

The method count is not reduced by hiding complexity behind strings. The generic surface remains constrained by strong IDs and closed domain types. There is no set(EntityRef, string, any), property-bag escape hatch, generic command bus, or mutable subsystem handle.

Complete definitions and immutable pin meaning

ComponentSpec

Contains the complete reusable shape: name, properties, provenance, symbol references, ordered pin definitions, pin semantics, and pin electrical attributes.

A successful call allocates canonical IDs and commits all rows atomically. A failed call changes nothing.

ModuleSpec

Contains template nets, ports, component occurrences, and internal connections. The complete graph is preflighted before the first append.

Module instantiation remains a separate canonical operation over the committed definition.

Committed component definitions own immutable pin definitions. PinDefId remains canonical identity for concrete pins, module templates, selected-part mappings, validation, and JSON. There is no public post-commit pin-definition setter and no accidental per-instance PinId electrical-attribute model.

Typed progressive updates

Some meaning is discovered after instantiation. Volt represents those changes with closed typed values instead of adding one root setter per field.

using ComponentUpdate = std::variant<
    SetComponentProperty,
    SetComponentElectricalAttribute,
    SelectPhysicalPart,
    SetSelectedPartElectricalAttribute,
    SetAssemblyIntent>;

using NetUpdate = std::variant<
    SetNetElectricalAttribute,
    AssignNetClass,
    MarkIntentionalStub>;

Every alternative is kernel-owned, typed, persistable, inspectable, and validated. New alternatives require a real EDA meaning and downstream contract; internal storage growth does not qualify.

Generic reads and free queries

get(Id)

One invalid-ID-checking primitive for canonical entities. Strong ID type selects the return type.

all<Id>()

One deterministic range primitive with range-for and size(), replacing per-table getter/count pairs.

Name/reference lookup, pin lookup, hierarchy traversal, origins, net-class resolution, adjacency, and other derived reads stay in volt::queries. They consume const Circuit&, return optional when absence is normal, and do not expose ConnectivityModel or HierarchyModel.

Bad circuit design remains diagnostic: unconnected required pins, single-pin nets, role conflicts, power-domain issues, insufficient ratings, unmapped physical readiness, and unrouted PCB implementation do not become structural exceptions.

Authoring remains concise

C++ kernel boundary

Circuit circuit;

auto led = circuit.define_component(
    ComponentSpec{
        .name = "LED",
        .pins = {
            PinSpec{.name = "A", .number = "1",
                    .terminal = Passive},
            PinSpec{.name = "K", .number = "2",
                    .terminal = Passive},
        },
    });

auto d1 = circuit.instantiate_component(
    led, ComponentInstanceSpec{.reference = "D1"});
auto gnd = circuit.add_net(
    NetSpec{.name = "GND", .kind = Ground});

circuit.connect(gnd,
    queries::pin_by_name(circuit, d1, "K").value());

Existing Python surface

import volt

design = volt.Design("led")
vcc = design.net("VCC", kind="power", voltage=5.0)
led_a = design.net("LED_A")
gnd = design.net("GND", kind="ground")

j1 = design.connector_1x02(ref="J1")
r1 = design.R("330 ohm", ref="R1")
d1 = design.LED(ref="D1")

vcc += j1[1], r1[1]
led_a += r1[2], d1["A"]
gnd += d1["K"], j1[2]

assert not design.validate().has_errors
design.write("led.volt.json")

The Python layer continues to provide handles, catalog helpers, operator sugar, and contextual errors. It lowers those operations into the same typed C++ contract and owns no alternate EDA meaning.

Boundary rules

The typed aggregate is the only public logical root. No borrowed subsystem facades or public subsystem mutators.
Reads are generic and typed. Use get(Id), all<Id>(), fundamental relationships, and free derived queries instead of table getters and counts.
Restoration remains private. The logical reader may apply its narrow validated restoration plan without exposing loader mechanics as authoring operations.
The architecture checker enforces admission. New storage-shaped methods, generic mutation handles, public submodel mutation, and Python-owned connectivity semantics fail the boundary checks.
GitHub controls dispatch. Only issues carrying both roadmap and ready with no native blockers may be selected autonomously.

Non-goals

  • No entity removal, erasure, undo, or mutable document-database model.
  • No command bus, ECS, event-sourced framework, generic repository, or plugin system.
  • No generic EntityRef traversal or mutation handle.
  • No transfer of logical/electrical meaning to Python, schematic, PCB, importers, or UI.
  • No redesign of the Python project or CLI experience in this migration.
  • No loss of typed electrical meaning or diagnostic depth to hit a method-count target.

Definition of done

  • One authoritative, small, typed Circuit aggregate API.
  • Complete failure-atomic component and module definitions.
  • Generic typed reads and representation-hiding free queries.
  • No public facades, passkeys, query-model leakage, raw restore hooks, or count/getter duplication.
  • Every rich semantic field, persistence relationship, diagnostic, Python workflow, schematic projection, PCB implementation, BOM path, and exporter remains correct.
  • The architecture checker prevents the storage-shaped surface from regrowing.
  • Every native sub-issue under #261 is complete and full CI passes.
