First Board LED PCB Example

This example is the smallest end-to-end Volt PCB authoring flow: a two-pin power header drives an LED through a resistor, then public Python APIs produce logical, schematic, validation, PCB JSON, and placement-only PCB SVG artifacts.

Authoring Flow

1. Logical circuit Create +3V3, LED_A, and GND, then connect J1, R1, and D1.
2. Selected parts Select real physical parts with object-owned footprint geometry, pin-to-pad mappings, and optional footprint-relative model_3d asset references on each component.
3. Projections Add the schematic and placement-only PCB as projections over existing logical connectivity.
4. Artifacts Validate and write deterministic JSON plus SVG previews from the kernel-owned model.

Files Written

pcb_led_board.volt.json Logical circuit JSON with component definitions, nets, selected parts, and pin mappings.
pcb_led_board.volt.schematic.json Schematic projection JSON; it visualizes existing nets but does not define connectivity.
pcb_led_board.svg Full schematic SVG preview with sheet metadata and title block.
pcb_led_board.volt.pcb.json Placement-only board JSON with outline, layer stack, mounting holes, placements, and pad resolution cache.
pcb_led_board.pcb.svg Deterministic board preview showing the board outline, placed footprints, and net overlays.
pcb_led_board.validation.json Combined logical, schematic, PCB readiness, and board validation report.
The board projection remains a physical implementation of the logical circuit. It places selected physical parts and cached footprint definitions; it does not create, merge, split, or otherwise mutate nets.
