Implementation plan

Convert Volt kernel headers into compiled libraries

Refactor the C++20 kernel from mostly header-only targets into compiled static libraries, starting with a tightly scoped PCB pilot. Keep behavior, validation, serialization, and public include compatibility unchanged unless a later phase explicitly calls out an API decision.

Date: 2026-06-01

Audience: agentic implementers and reviewers

Required execution skill: use superpowers:subagent-driven-development or superpowers:executing-plans task by task.

Primary verification: cmake --preset dev, cmake --build --preset dev, ctest --preset dev.

The reason for doing this at all.

Goal

Make the kernel compile once per library instead of recompiling roughly 25k lines of inline implementation in every test, example, benchmark, and Python binding translation unit.

Must

Preserve behavior

Golden fixtures, diagnostics, thrown exception types, and serialized output stay stable.

Must

Preserve includes

Existing consumers of umbrella headers such as volt/pcb/board.hpp continue to compile.

Measure

Prove the win

Record clean-build and targeted incremental rebuild timings before and after the PCB pilot.

Decisions to lock before implementation starts.

Architecture Decisions

Decision

Move bodies, not semantics

This is a build-structure refactor. Do not change kernel invariants, diagnostics, serialization field order, schema names, or public behavior.

Decision

Compiled targets use public usage requirements

Public include directories, C++20 compile features, and public dependency targets must stay PUBLIC. Use PRIVATE only for implementation-only dependencies.

Decision

Static libraries must be PIC-safe

New static kernel libraries need position-independent code so they can link into the Python extension _volt on Linux and wheel builds.

Decision

JSON encapsulation is separate

Hiding nlohmann::json from Volt::IO is valuable, but it is a source-breaking API change for parsed-JSON reader overloads. Do it as its own phase after the compiled-library mechanics are proven.

Convert volt_pcb to a compiled static library

Replace add_library(volt_pcb INTERFACE) with sources, keep usage requirements PUBLIC, link Volt::Circuit publicly, apply project warnings, sanitizers, coverage, and PIC. 

add_library(
    volt_pcb
    board.cpp
    board_geometry.cpp
    board_layers.cpp
    board_primitives.cpp
    board_ratsnest.cpp
    board_validation.cpp
    footprints.cpp
)
set_target_properties(volt_pcb PROPERTIES POSITION_INDEPENDENT_CODE ON)
target_compile_features(volt_pcb PUBLIC cxx_std_20)
target_link_libraries(volt_pcb PUBLIC Volt::Circuit)
target_include_directories(volt_pcb PUBLIC
    $<BUILD_INTERFACE:${PROJECT_SOURCE_DIR}/include>
    $<INSTALL_INTERFACE:include>
)
volt_apply_project_warnings(volt_pcb PRIVATE)
volt_apply_sanitizers(volt_pcb)
volt_apply_coverage(volt_pcb)

Follow-on conversion order after the PCB pilot is reviewed.

Library Rollout

1. Circuit

Move bodies from circuit.hpp and sibling headers

Do not decompose the Circuit class by responsibility in this pass. Move ordinary bodies into src/circuit/*.cpp and preserve all mutation-boundary invariant checks exactly.

2. Schematic

Split schematic.hpp, then compile bodies

Mirror the PCB split: sheet metadata, sheet content, endpoints, symbols-instances, and Schematic mutation APIs. Keep umbrella compatibility through volt/schematic/schematic.hpp.

3. Authoring

Compile ergonomic helpers

Move concrete component-library, connection-helper, and reference-designator bodies. Keep this layer as syntax over kernel-owned state only.

4. IO

Compile readers and writers before hiding JSON

First move implementation out of the reader and writer headers. Then choose whether to remove parsed-JSON overloads, deprecate them, or move them to an optional adapter. Only after that may nlohmann_json::nlohmann_json become a private dependency.

5. KiCad

Compile adapter export code

Keep the adapter outside Volt::Volt. Link only the circuit and schematic targets it actually consumes.

What not to blur into this refactor.

Out of Scope

No

No semantic redesign

Do not split the Circuit god class or change ownership of EDA meaning.

No

No validation changes

Do not move structural integrity checks out of mutation APIs or diagnostics into core mutation paths.

No

No broad install/export work

C++ package installation is a separate project because current install rules only cover the Python extension.

The failure modes most likely to bite.

Risk Register

Risk Symptom Mitigation
Duplicate symbol definitions Linker errors after moving an inline body into .cpp. Remove inline from declarations that now have exactly one compiled definition.
Lost public usage requirements Consumers fail to find headers or lose C++20 features. Use PUBLIC for include dirs, compile features, and public target dependencies.
Private helper made too private IO or SVG code no longer compiles after PCB helper movement. Search for external volt::detail callers before hiding helpers.
Strict warnings surface for library code New -Wconversion, -Wsign-conversion, or -Wshadow failures. Fix warnings surgically in moved implementation code. Do not soften the warning policy.
Static library not suitable for Python module Linux or wheel builds fail while linking _volt. Set PIC on all new static kernel libraries and keep Python dev preset in verification.
Coverage check silently excludes new sources Coverage target runs but gates the wrong compiled target set. Update scripts/check-build-config.py and coverage target dependencies with new libraries.
When a phase is done.

Definition of Done

Pilot done

  • volt_pcb is a compiled static library with real implementation sources.
  • board.hpp remains an umbrella include and existing consumers compile unchanged.
  • Non-trivial PCB and footprint bodies live in src/pcb/*.cpp.
  • Full ctest --preset dev passes.
  • Clean-build and incremental rebuild timings are reported.

Whole refactor done

  • All six former header-only targets are compiled static libraries.
  • Public headers retain only declarations, templates, constants, and trivial accessors.
  • nlohmann_json is private only if public JSON overloads have been removed or isolated.
  • CI-relevant dev, ASAN, coverage, Python, docs, and examples still pass.
  • The final summary lists behavior-preservation evidence and build-time deltas.
