VOL-216 BOM Projection Implementation Plan

Goal: Emit deterministic bom.json and bom.csv from kernel-owned BOM intent, with DNP, per-instance selected-part overrides, alternates, sourcing snapshots, and stable BOM-readiness diagnostics.

Architecture

DNP belongs in DesignIntent as typed component-instance intent. Approved alternates belong on selected/orderable part identity, not Python-only metadata. BOM projection is a C++ kernel read model over Circuit plus an optional design-owned MPN sourcing snapshot; Python only exposes and writes that projection into project bundles.

Tasks And Verification

  1. Kernel intent and selected-part shape. Add failing C++ tests for component DNP intent, selected-part approved alternates, and logical JSON round-trip. Implement DesignIntent, Circuit, logical reader/writer, and Python binding plumbing. Verify with logical_circuit_writer_test, logical_circuit_reader_test, and circuit_test.
  2. BOM projection. Add failing C++ tests for deterministic grouping by manufacturer/MPN/package, DNP visibility, alternates visibility, per-instance selected-part override behavior, reference sorting, and MPN-level sourcing merge. Implement focused BOM model/writer files under include/volt/circuit and src/circuit. Verify with new tests/circuit/bom_test.cpp.
  3. BOM readiness diagnostics. Add failing C++ tests for populated instance missing selected part, missing explicit DNP intent, and malformed alternate compatibility. Implement validate_bom_readiness, stable BOM diagnostic codes, category/catalog exposure, and Python bindings. Verify with validation and diagnostics catalog tests.
  4. Python authoring API. Add failing Python tests for component.dnp(), select_part(..., approved_alternate_mpns=...), design.validate_bom_readiness(), and design.bom(). Implement thin wrappers over kernel state. Regenerate Python API docs if the public surface changes.
  5. Project bundle output. Add failing project-framework golden tests proving bom/bom.json and bom/bom.csv appear in manifest.volt.json, are deterministic, include DNP/alternates/overrides, and merge optional design-owned sourcing by MPN. Implement bundle writing without introducing VOL-188 CPL or assembly-bundle scope.
  6. Final verification and PR. Run targeted C++ and Python tests, then broader available tests. Push feat/vol-216-bom-projection-dnp-overrides-alternates, open a ready PR titled with VOL-216, dispatch a thermonuclear review thread, address actionable findings, and leave review findings on the PR.

Out Of Scope

No CPL, full assembly bundle, fab-house submission, registry, lockfile, trust-tier, rich distributor integration, or alternate BOM shape for VOL-188. Sourcing remains a design-owned snapshot keyed by MPN and never enters part hashes or library code.
