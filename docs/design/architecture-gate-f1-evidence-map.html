Review delta, not a permission slip

The evidence baseline lives in tests/architecture/refactor_inventory_baseline.json. The checker compares public snapshot hashes, Python contracts, canonical artifact shapes, ownership evidence, exact offline fixtures, semantic summaries and raw byte hashes. Drift fails so a reviewer sees the change. Updating the evidence records the reviewed delta; it does not make the change architecturally valid.

Separation of roles

Evidence and invariants

Compact review evidence

  • Public API snapshot files show the declaration-level delta; their hashes bind the inventory to that exact content.
  • Python exports and exception metadata preserve public binding contracts without counting private binding implementation.
  • Artifact kinds, manifest fields and exact fixtures preserve canonical and offline compatibility.
  • Goldens freeze normalized JSON shape plus raw byte length and SHA-256.

Objective invariants

  • Retired APIs and storage shells may not reappear.
  • Ownership crossings, subsystem back-references and public raw storage exposure fail directly.
  • Root declarations and borrowed lifetimes retain their bounded shapes.
  • Existing Python-only connectivity, hidden dependency, typed-error and EntityRef traversal boundaries remain checked.
Removed by #328

What no longer belongs here

Mutable approval machinery

The architecture checker no longer parses or classifies friend declarations, keeps friend exception tables, or treats repository-owned friend data as an architecture decision. Existing C++ friend declarations remain product code for later focused work.

High-churn implementation inventory

Direct caller strata, private binding-call counts, IO and test filename counts, catalogue and example call counts, CLI/project implementation shapes, baseline commit metadata and future-slice ownership tables are removed. GitHub issues and native dependencies own planning.

Stop rule

If an evidence file changes, inspect the product delta it exposes. If an objective invariant fails, fix the forbidden state; do not add a waiver entry.
