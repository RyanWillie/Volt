## What Ucamco X2 Attributes Look Like

Per the Gerber Layer Format Specification (rev 2024.05), X2 attributes are **extended commands** in the `%TF...%` / `%TA...%` / `%TO...%` format:

### File Attributes (`%TF.*%`) — placed in header, before aperture definitions

```gerber
%TF.GenerationSoftware,Volt,Volt EDA,1.0*%
%TF.CreationDate,2026-04-17T12:00:00Z*%
%TF.ProjectId,MyBoard,<uuid>,1.0*%
%TF.FileFunction,Copper,L1,Top*%         ← layer role
%TF.FilePolarity,Positive*%
%TF.Part,Single*%                        ← always "Single" for a single PCB
```

### FileFunction values per layer type:

| Layer | FileFunction value |
|---|---|
| Top Copper | `Copper,L1,Top` |
| Inner Copper N | `Copper,L{N+1},Inr` |
| Bottom Copper | `Copper,L{total},Bot` |
| Top Soldermask | `Soldermask,Top` |
| Bottom Soldermask | `Soldermask,Bot` |
| Top Silkscreen | `Legend,Top` |
| Bottom Silkscreen | `Legend,Bot` |
| Top Paste | `Paste,Top` |
| Bottom Paste | `Paste,Bot` |
| Board Outline | `Profile,NP` (non-plated profile) |

### FilePolarity values:
- Copper, Legend, Paste, Profile → `Positive`
- Soldermask → `Positive` (openings = dark regions; the mask itself is inverted by fab)

## Implementation Plan

### Step 1: Add X2 attribute helper to `GerberWriter` (gerber.rs)

Add a method to emit raw X2 extended attributes:

```rust
/// Emit a file attribute (`%TF.{key},{values}*%`).
pub fn file_attribute(&mut self, key: &str, values: &[&str]) {
    self.buf.push_str("%TF.");
    self.buf.push_str(key);
    for v in values {
        self.buf.push(',');
        self.buf.push_str(v);
    }
    self.buf.push_str("*%\n");
}
```
